# Recipe: Bootstrap Desktop Codex Through Tailscale

This is the intended product flow for connecting MobileClaw to a clean desktop Codex install.

## Goal

The user should not manually clone MobileClaw on the computer or copy bridge files by hand.

Expected user state:

- Codex CLI is already installed and authenticated on the computer.
- MobileClaw is installed on the phone.
- Tailscale is installed on both phone and computer.
- The user knows the computer's Tailscale IP or MagicDNS name.

MobileClaw then pushes a small desktop bundle to the computer:

- MobileClaw Codex bridge script.
- A local Codex plugin package.
- Install/start scripts.
- Usage instructions.

## Required Transport

Tailscale IP alone is not enough to write files to the computer. MobileClaw needs one of these transports:

- Tailscale SSH, preferred.
- Normal SSH over the Tailscale IP.
- A one-time desktop bootstrap command which opens a temporary pairing endpoint.

The first implementation should use Tailscale SSH/SSH because it is explicit, secure, and easy to reason about.

## Mobile UX

In Settings > Codex Bridge, add a "Desktop Bootstrap" flow:

1. User enters:
   - Tailscale host: `100.x.y.z` or `desktop-name.tailnet.ts.net`
   - SSH user
   - Optional SSH port, default `22`
   - Bridge token, generated by MobileClaw by default
2. MobileClaw checks SSH connectivity.
3. MobileClaw uploads the bundled desktop package to:
   - `~/.mobileclaw/codex-desktop/`
4. MobileClaw runs:
   - `bash ~/.mobileclaw/codex-desktop/install.sh`
5. MobileClaw saves:
   - Bridge URL: `http://<tailscale-host>:52734`
   - Token
   - Default cwd/model/provider/sandbox settings
6. MobileClaw tests:
   - `GET /health`

## Desktop Bundle Layout

The package pushed by MobileClaw should contain:

```text
codex-desktop/
  install.sh
  start_bridge.sh
  mobileclaw-codex-bridge.py
  README.md
  plugins/
    mobileclaw-desktop/
      .codex-plugin/
        plugin.json
      skills/
        mobileclaw-desktop/SKILL.md
  marketplace.json
```

## Install Script Responsibilities

`install.sh` should:

1. Verify `python3` exists.
2. Verify `codex` exists.
3. Create `~/.mobileclaw/codex-desktop/`.
4. Copy bridge files into that folder.
5. Install or register the local Codex plugin marketplace when supported.
6. Write a default config file:
   - `~/.mobileclaw_codex_bridge.json`
7. Print the bridge start command.

It should not overwrite the user's Codex auth or global config.

## Start Script Responsibilities

`start_bridge.sh` should start the bridge with:

```bash
CODEX_BRIDGE_HOST=0.0.0.0
CODEX_BRIDGE_PORT=52734
CODEX_BRIDGE_TOKEN=<generated-token>
python3 ~/.mobileclaw/codex-desktop/mobileclaw-codex-bridge.py
```

For a first version, MobileClaw may start it in the SSH session with `nohup`.

Later versions can install a launchd plist on macOS or a systemd user service on Linux.

## Security Rules

- Generate a random token on the phone.
- Never expose the bridge without bearer token auth.
- Prefer Tailscale-only access.
- Do not open public internet ports.
- Do not request or copy Codex auth tokens from the computer.
- Default cwd should be explicit, not the whole home folder.
- MobileClaw must expose a stop button for active Codex processes.

## Implementation Phases

Phase 1:

- Bundle files in the APK assets.
- Add manual SSH bootstrap screen.
- Upload files through Tailscale SSH/SSH.
- Run `install.sh`.
- Start bridge with `nohup`.

Phase 2:

- Add QR/pairing flow.
- Add service installation for macOS/Linux.
- Add bridge update/version check.
- Add plugin version check.

Phase 3:

- Add relay fallback for users who cannot enable SSH.
